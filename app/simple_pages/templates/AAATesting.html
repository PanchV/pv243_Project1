What is AAA?

AAA is a pattern that is present in Unit Testing in Python that stands for Arrange, Act, Assert. But first, what exactly is a Unit Test? The unit test is a testing framework that supports test automation, sharing of setup and shutdown code for tests, aggregation of tests into collections, and independence of the tests from the reporting framework. The AAA pattern is one where we divide the test method into three sections which each have their own special role.

  • Arrange
    • Have the code properly set up for the test. Create any necessary objects for a mock setup if necessary.
  • Act
    • Invoke the method to be tested.
  • Assert
    • Check whether the expectations were met.

Why It is important


As we mentioned in the earlier Python-Flask article on this site, it is crucial that testing is in place. The absolute worst thing in the devops world to do is to have one bad programmer merge a branch with faulty or broken code with the main branch. Testing combats this by making sure that every single time someone tries to push their branch to the master branch, their branch will have to pass tests that will ensure that the code they've just written won't break the main program. AAA testing is very important because it gives a standard format for developing new tests, which makes it easier to implement our tests throughout our code and can help us test elements and methods we never thought possible before.
