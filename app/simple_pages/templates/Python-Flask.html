What is Python-Flask?

Python Flask is a web framework. It is a Python module that lets you develop web applications really easily. This means that flask provides you with all of the libraries tools, packages, and software and that are necessary to build a web application. In fact, this web page and whole project were created using Flask! Flask is known as a micro-framework due to the fact that it has very few external libraries and dependencies.

Testing With PyTest

When developing with a team of other devops engineers, it is crucial that testing is in place. The absolute worst thing in the devops world to do is to have one bad programmer merge a branch with faulty or broken code with the main branch. What will tend to happen is everyone will continue to make commits and merges after this one faulty branch. Several merges later, surprise surprise, the code no longer works. even worse, they will never know whose bad merge broke the source code. Lots of time and resources will be spent trying to identify the problem. With testing, every single time someone tries to push their branch to the master branch, their branch will have to pass tests that will ensure that the code they've just written won't break the main program. This webpage has mutiple tests that can be found here:
  • Testing Simple Pages
  • Testing Context Processes
  • Testing the configuration of the application

Creating And Testing Simple Pages

Simple pages is a structure that allows us to maintain all of the pages in a website effectively and effeciently. When first approaching web development, it may seem intuitive to simply create multiple html files that contain all of your content and then have them pull up as the user navigates the web page. However, it is much better practice to have one main base html page that "calls" or points to your other pages. These other pages "extend" on your base. Within your base page, you can have all of teh elements that will be common throughout your site, such as a navigation bar. This is exactly how they are set up on this website. One base.html files calls other html files(Git.html, Docker.html) which all extend on base.html. All of these simple pages are tested with the following: https://github.com/PanchV/pv243_Project1/blob/master/tests/simple_pages_test.py

Files Explained

  • docker_flask* <- Project Root Directory
    • .github* <- Contains yaml files that deploy the project
      • workflows* <- Sub-folder that contains yaml files that deploy the project
        • dev.yml <- Development environment deployment to test updates and changes
        • heroku_awake_dev.yml <- Github action to keep heroku dev application awake, otherwise it would go to sleep if it does not receive any web traffic
        • heroku_awake_prod.yml <- Github action to keep heroku prod application awake, otherwise it would go to sleep if it does not receive any web traffic
        • prod.yml <- Production environment deployment to keep a version of the app always running
    • app* <- contains application related files such as the html pages, related style sheets, js scripts, and images
      • context_processors* <-Contains utility text initialization file
        • __init__.py <- Initializes utility text at the bottom of the page
      • simple_pages* <-contains individual content pages that are called and Flask blueprint
        • templates* <- Individual content pages that are called
          • CI-CD.html <- Continuous Integration and Continuous Deployment content page
          • Docker.html <- Docker content page
          • Git.html <- Git content page
          • index.html <- Home content page
          • page4.html <- Blank page from template, not used
          • Python-Flask.html <- Python Flask content page
        • __init__.py <- Flask blueprint
      • static* <- Holds static elements that are used such as images, style sheet, and scripts
        • css* <- Holds Cascading Style Sheet
          • style.css <- Style sheet used to make more dynamic elements in HTML
        • images* <- holds the logo and other images in image carousel
        • js* <- holds the javascript file
          • scripts.js <-Scripts to make interactive HTML elements
      • templates* <- Holds main HTML file
        • base.html <- Main HTML file that calls other simple pages to display
      • __init__.py <- Creates Flask web application
      • run.py <- Allows Green Unicorn(Python Web Server Gateway) to serve the app in production
    • tests* <- contains all the tests for this application
      • conftest.py <- Testing the configuration of the application
      • context_processes_test.py <- Testing Context Processes
      • simple_pages_test.py <- Testing Simple Pages
    • docker-compose.yml <- Orchestration file that contains the configuration to develop locally, overrides the Dockerfile to run the flask dev server instead of running the gunicorn server that is used for hosting on Heroku
    • Dockerfile <- File used to create an image to run in a container that runs the program
    • heroku.yml <- Builds the Docker image on Heroku
    • pytest.ini <- initilizes and configues the pytest
    • readme.md <- Generates the html summary seen at the end of a repo, used to convey info about the project
    • setup.py <- Installs necessary dependencies and packages
* - indicates a directory
