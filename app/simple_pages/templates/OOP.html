What is Object-Oriented Programming?

Object-Oriented Programming, often referred to as OOP, is a framework that helps organize the design of software around units called Objects rather than around the functions and the logic of the program. We will do a further deep dive of what exactly an 'Object' is further below. For now all we need to focus on is the fact that this approach is perfect for complex projects that tend to be very large with many moving parts that are updated frequently. Each of the items below will walk through the building blocks, and the main principles of Object-Oriented Programming, we will also see how our calculator program utilizes/demonstrates Encapsulation, Inheritance, Polymorphism, and Abstraction.

Glossary of Key OOP Terms:

Instantiation


In OOP, instantiation refers to creating an Object, which is an 'instance' of a given class. In our calculator project, we have a Calculator class with a result property and multiple methods. However, this is just a blueprint. We create a calculator object by instantiating the Calculator class in the following way:
calculator = Calculator()

Object


An object is a software package or unit that contains variables and functions. The variables define the states that the object can exist, and the functions define the behavior of the object. In the above example, we created a calculator object through instantiation.

Class


A class is the blueprint or plan from which objects are created. Classes provide default values for variables and implementations of functions. Classes support 'Inheritance' and hence can be utilized to maintain a class hierarchy. Here is our calculator class: 
class Calculator:
    """ This is the default result property"""
    result = 0

    def add(self, value_1):
        """ This is the add method"""
        self.result = self.result + value_1
        return self.result

    def subtract(self, value_1):
        """ This is the subtract method"""
        self.result = self.result - value_1
        return self.result

    def multiply(self, value_1, value_2):
        """ This is the subtract method"""
        self.result = Multiplication.multiply(value_1, value_2)
        return self.result

    def divide(self,value_1, value_2):
        """This is the divide method"""
        if (value_2 == 0):
            self.result = "Divide By Zero Error!"
        else:
            self.result = value_1/value_2
        return self.result

    def get_result(self):
        """ This is the get result method"""
        return self.result

Namespace


A namespace is a collection of symbolic names/variables defined by the author of a program and the object that the symbolic name corresponds to. We can think of a namespace as a dictionary where the words are the object names and the definitions are the objects themselves.

Constructor


In OOP, a constructor is specific method that is called to create(initialize) an object. The constructor will prepare a new object that will accept arguments to be used by the constructor set required variables. It is worth noting that a constructor is not a method because it has no return type. In our calculation class, we use a constructor to give values to an object: 
def __init__(self, args_tuple: tuple):
        """ constructor """
        self.values = Calculation.convert_args_to_float_tuple(args_tuple)

Fixture


In the context of tests in OOP, Fixtures are functions that provide an environment for tests. They run before each test function that they are applied to and feed input data to test. Here is an example from our configuration test: 
@pytest.fixture()
def application():
    """This makes the app"""
    application = create_app()
    application.config.update({
        "TESTING": True,
    })
    yield application

Type Hint


A Type Hint is a formal way to statically indicate the type of a value within your Python code. The main purpose of type hinting is to give developers a way to known which object types are associated with which variables.

Type Cast


Type Casting is the method to convert the variable data type into a certain data type in order to the operation required to be performed by users. In implicit type casting, Python converts data type into another data type automatically. In explicit type casting, the developer needs to manually convert the data type of a variable into another data type.

Unit Test


The unit test is a testing framework that supports test automation, sharing of setup and shutdown code for tests, aggregation of tests into collections, and independence of the tests from the reporting framework.

Static


In object-oriented programming, static is a term that is applied to functions, methods, and member variables and basically means that a certain function, method, or member variable can be accessed directly from the constructor and doesn't have to come from the instance of the object.

Instance Method


An instance method is a method in a class that takes a self as a parameter and points to an instance of the class. Through the self parameter, instance methods can freely access attributes and other methods on the same object. This gives them a lot of power when it comes to modifying an object’s state. Here is an example from our calculator class: 
def add(self, value_1):
        """ This is the add method"""
        self.result = self.result + value_1
        return self.result

Instance Property


A property that is declared in a class is an instance property. This means that there is a copy of that property's value in every instance of that class. This can be handy when we have a certain value that we want to make sure that every single instance has.

Static Method


A static method is a method that is bound to a class, not its object. These methods do not require a class instance creation and can be called without an object for that class. The use case for static methods is when we need to create Utility methods as they aren’t tied to an object's lifecycle. Here is a static method from our Calculations class. 
@staticmethod
def clear_history():
        """Clear history list"""
        Calculations.history.clear()

Static Property


A static property, much like a static method, is simply a property that is belongs to the class, not just an instance of the class. This is used when a property needs to be unique and there can only be one of it. If a property needs to have independent values across multiple instances, then the property can’t be static. Here is a static property from our calculator class: result = 0

Encapsulation


In OOP, encapsulation is the bundling of data, along with the methods that operate on that data, into a single unit. This principle is really great because it allows us to hide specific information and control access to the internal state of the object. This lets us create "getter" and "setter" methods which retrieve and change methods respectively. In the Calculator program, we see encapsulation when we made our Calculator class. We bundled the add, subtract, multiply, and divide methods

Inheritance


Inheritance is the mechanism where you derive one class from another class to develop a hierarchy of classes that share a common set of attributes and methods. The class where the attributes and methods are derived from is known as the parent class and the class that receives the attributes and methods is known as the child class. In the Calculator program

Polymorphism


Polymorphism is the concept that describes how multiple objects with differing attributes can be accessed from the same interface. In Python, Polymorphism lets us define methods in the child class that have the same name as the methods in the parent class. In the Calculator program

Abstraction


Abstraction is the concept of object-oriented programming that “shows” only essential attributes and “hides” unnecessary information. The primary purpose of this principle is to hide unnecessary information from the end-user In the Calculator program
