What is Docker?

Docker is an open source platform that allows DevOps Engineers build, deploy, and manage applications into components called "containers". It allows developers to separate the applications that they run from the infrastructure. This allows software to be deployed quickly compared to traditional methods. With how Docker is set up, one can drastically reduce the delay between writing software and implementing it into production.

Containers

A Container is an isolated environment where code and all of its dependencies and libraries can be packaged up together so that an application can be run in said environment. It is possible to run multiple containers on the same machine, and in turn, all of these containers share the Operating System Kernel, and each container runs as an isolated process. This makes containers extremely popular for running distributed applications. It is executed as a container "Image".

Images

A Docker Container Image is simply a file that is used to execute the code and related dependencies in a container. In other words, they are the instructions or plan to build up a container. An image is an instance or a snapshot of a Docker container at a specific point in times. Docker images are also particularly useful because they can't be changed as they are immutable which is a useful feature when testing newly written software.

Docker Commands

Some common commands and what they do:

  • - version: lets you see your most recently installed version of docker
  • images: lets you see all the docker images stored locally
  • pull: lets you get images from the docker repository to work on locally
  • push: lets you share your local docker image to a repository
  • build: lets you create a docker image from a specific docker file
  • run: lets you create a container from an image
  • ps: lets you view the currently running containers
  • rm: lets you delete a stopped container
  • stop: lets you stop a running container(gives container time to shut down gracefully)
  • kill: lets you stop a container's executions immediately(used when container is taking too long to shut down)
  • commit: lets you make a new image of a changed container locally
