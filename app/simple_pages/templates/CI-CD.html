What is CI-CD?

CI-CD stands for Continuous Integration and Continuous Deployment, sometimes referred to as Continuous Delivery. This principle is integral to DevOps and it refers to the practice of building, testing, and deploying code continuously. CI-CD is meant to combat the issues that persist with traditional application development methods, where issues with integrating new code are common. CI-CD avoids this by testing code as soon as it gets pushed, and if it passes the tests, it will merge, then automatically release to the repository, and then deploy to production.

GitHub Actions

Github Actions allow us to implement the process of CI-CD into projects such as this one. They actually go a little bit further than DevOps because they allow certain events such as a pull request to trigger workflows. Github does this via jobs, each of which run inside their own virtual machine/container known as a runner. Jobs are sets of steps that reside in the yaml file that execute on the same runner.
