What is Pylint?

Pylint is a code analysis tool that's there to help us enforce a standard for coding, detect code smells (a subjective characteristic in the source code of a program that possibly indicates a deeper problem), identify programming errors, and also offers suggestions for refactoring code in a simple way. Below are examples of how pylint helped us improve our program:

The triple quotes which help us write comments in our code are actually read as docstrings and python doesn't actually consider them comments. These can be removed as they were just put in to the code to help demonstrate the AAA testing pattern.

It may seem like many of these suggestions from pylint are pointless and actually serve no purpose to fixing the actual code, but it is important to stick to a standard convention when writing code so that other developers can understand and follow it.
